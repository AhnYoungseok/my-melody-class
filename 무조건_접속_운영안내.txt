현실적으로 가장 안정적인 구성

1순위: GitHub Pages 공개 주소
- 와이파이가 달라도 접속됩니다.
- 교실 컴퓨터를 켜두지 않아도 됩니다.
- 학생은 인터넷만 되면 접속합니다.
- 주소 예시: https://아이디.github.io/저장소이름/

2순위: 교실 컴퓨터 로컬 서버
- 인터넷이 불안정하거나 GitHub 접속이 막힐 때 예비로 씁니다.
- 이 경우에는 같은 Wi-Fi 조건이 필요합니다.

정말 '무조건'에 가장 가까운 운영법:
1. GitHub Pages 주소를 먼저 준비합니다.
2. 같은 파일을 USB에도 담아 갑니다.
3. 교실 컴퓨터에는 `교실_안정운영_패키지.zip`도 같이 가져갑니다.
4. 인터넷이 되면 GitHub Pages 주소를 씁니다.
5. 인터넷이 막히면 교실 컴퓨터 로컬 서버로 바꿉니다.

절대 보장할 수 없는 경우:
- 학생 기기 자체가 인터넷이 안 되는 경우
- 학교에서 GitHub Pages 접속을 차단한 경우
- 기기 브라우저가 소리 재생을 막아 재생 버튼을 직접 눌러야 하는 경우

그래도 이 구성은 수업 현장용으로 가장 실패 가능성이 낮습니다.
