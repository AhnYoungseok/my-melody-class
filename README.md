# 인터넷 공개용 배포 파일

이 폴더는 와이파이가 달라도 접속할 수 있게 GitHub Pages에 올리는 용도입니다.

## 가장 중요한 파일

- `index.html`: 학생들이 접속할 실제 앱
- `404.html`: 주소를 잘못 눌러도 앱으로 돌아오게 하는 예비 파일
- `.nojekyll`: GitHub Pages가 파일을 그대로 내보내게 하는 설정 파일
- `교실에서_인터넷주소로_쓰기.txt`: 교실에서 바로 볼 안내문

## GitHub Pages로 공개하는 방법

1. GitHub에 로그인합니다.
2. 새 저장소를 만듭니다. 예: `my-melody-class`
3. 이 폴더 안의 파일 4개를 저장소에 업로드합니다.
4. 저장소의 `Settings`로 들어갑니다.
5. 왼쪽 메뉴에서 `Pages`를 누릅니다.
6. `Build and deployment`에서 `Deploy from a branch`를 고릅니다.
7. Branch는 `main`, 폴더는 `/root`를 고르고 저장합니다.
8. 잠시 기다린 뒤 `https://아이디.github.io/저장소이름/` 주소로 접속합니다.

## 수업에서 쓰는 방식

- 교실 컴퓨터와 아이패드가 서로 다른 와이파이에 있어도 됩니다.
- 학생들은 모두 같은 공개 주소로 접속합니다.
- 학생 작업은 각 기기의 브라우저에 저장됩니다.
- 인터넷 자체가 끊기면 접속할 수 없습니다. 이 경우에는 `교실_안정운영_패키지.zip`의 로컬 서버 방식을 예비로 씁니다.
